# /run-all

프리셋 하네스의 **전체 파이프라인**을 순차 실행합니다. 각 단계 내 루프는 그대로 유지되며, 단계 간 전환 시 프로파일에 따라 사용자 승인 게이트가 붙습니다.

## 수행 절차

1. **사전 점검**
   - `harness.config.json` 존재 확인. 없으면 `/init-harness` 선행 안내
   - 이미 진행 중인 단계가 있으면 이어받기/재시작 중 선택 요청 (AskUserQuestion)

2. **순차 실행**
   - 단계 1: `/analyze` 로직 실행
   - 단계 2: `/design` 로직 실행
   - 단계 3: `/develop` 로직 실행
   - 단계 4: `/verify` 로직 실행 (종합 검증)
   - 단계 5: `/test` 로직 실행

3. **단계 전환**
   - 각 단계 완료 후 프로파일 확인:
     - `minimal` → 요약만 출력, 자동 진행
     - `standard` → 요약 출력, 자동 진행 (실패는 루프)
     - `strict` → 사용자 승인 게이트 필수 (AskUserQuestion) — 진행 / 중단 / 해당 단계 재실행 중 선택
   - 각 단계 phase 커밋은 git-master 호출 (사용자 승인 후)

4. **중단 조건**
   - 임의의 단계에서 `nextAction: block` (최대 반복 횟수 초과)
   - 사용자가 승인 게이트에서 중단 선택
   - 중단 사유를 요약 리포트로 제시

5. **완료 리포트**
   - 전체 단계별 통과 여부·루프 횟수·소요 요약
   - 생성된 주요 파일·커밋 해시
   - 잔존 보완 지시 있으면 리스트업

## 제약

- 각 단계는 이전 단계 `verify-<stage>.json`의 `pass: true`를 필수 조건으로 함
- 병렬 실행은 단일 단계 내 에이전트 수준에서만 (단계 자체는 순차)
- 사용자 중단 요청 시 진행 중 작업 안전하게 저장 후 종료
